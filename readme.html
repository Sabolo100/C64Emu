C64 BASIC Emulator README

Ez a dokumentum a README.md webes nezetet adja.

Alap hasznalat

  • Sorszamozott programsor: 10 PRINT "HELLO"
  • Sor torlese: 10
  • Listazas: LIST vagy LIST 100-300
  • Futtatas: RUN vagy RUN 100
  • Folytatas STOP/BREAK utan: CONT
  • Uj program: NEW

Programozasi mod

  • A sorszammal kezdo sorok csak mentesre kerulnek, nem futnak le Enterre.
  • A kod futtatasa kulon RUN paranccsal tortenik.
  • SHIFT+ENTER automatikusan felajanlja a kovetkezo sorszamot (+10).
  • A kepernyotartalom + program allapot automatikusan mentodik localStorage-ba, igy README utan visszaall.

Kepernyo mozgas

  • ArrowUp / ArrowDown: programsorok kozti lepkedes szerkeszteshez.
  • Ha a legutolso programsoron vagy, ArrowDown visszavisz az also parancssorra.
  • LIST utan egy sorszamozott sorra kattintva szerkesztesre betolt.
  • Programsor-szerkesztes kozben, ha nem programsorra kattintasz, visszalep a parancssorra.
  • Szerkesztes Enterre helyben frissul, nincs kulon duplikalt uj sor.
  • Alt+ArrowUp / Alt+ArrowDown: scroll a kepernyobufferben.
  • PageUp / PageDown: gyorsabb scroll.
  • Eger gorgetes is mukodik a kepernyon.

Nezet valtas

  • MAX NEZET: nagyobb befoglalo terulet, valtozatlan font meret.
  • NORMAL NEZET: visszavalt az alap meretre.
  • Gyorsbillentyu: F9.

Publikus deploy (Vercel)

  • Tedd fel ezt a projektet GitHub repositoryba.
  • Vercelben: Add New Project, majd importald a repositoryt.
  • Ha nem repository gyokerben van a projekt, Root Directory: ChatGPT Codex/C64 Emulator.
  • Framework: Other, build command nem szukseges.
  • A gyoker URL automatikusan az index.html-re mutat a vercel.json miatt.

AI local inditas (3131)

  • Futtasd: ./start-vercel.sh
  • Ez fixen a 3131 portra inditja a Vercel dev servert.
  • Nyisd meg: http://localhost:3131

Tarolas (lokalis)

  • SAVE "PROG",8: program mentese localStorage-ba.
  • LOAD "PROG",8: program betoltese.
  • LOAD "*",8,1: utolso mentett program betoltese.
  • VERIFY "PROG",8: program osszevetese a mentettel.

Tamogatott BASIC elemek

  • Programkezeles: LIST, RUN, NEW, CONT, HELP, TESTPACK
  • I/O es csatorna: OPEN, CLOSE, CMD, PRINT#, INPUT#, GET#
  • Vezerles: IF THEN ELSE, GOTO, GOSUB/RETURN, ON ... GOTO/GOSUB, FOR/NEXT, STOP, END
  • Adatkezeles: DATA, READ, RESTORE, DIM, DEF FN, LET, CLR
  • Rendszer: POKE, WAIT, GET, SYS
  • Grafika (C128 stilus is): GRAPHIC, SCNCLR, COLOR, DRAW, BOX, CIRCLE, PAINT, valamint PLOT, LINE, RECT, CLSG
  • Kepernyotorles: CLS (text), CLSG (grafika)
  • AI: footer AI gomb feladatbol BASIC program generalasra.

Parancs ellenorzes

  • A lista az emulator parseren (executeImmediate + executeStatement) lett ellenorizve.
  • Kulon vannak azonnali parancsok (pl. LIST, RUN) es programsor-utasitasok.

Parameter formatumok

  • Valtozonev: numerikus (A, I1) vagy sztring (N$).
  • Tomb: DIM A(10), DIM M(5,5), indexeles: A(I), M(I,J).
  • DIM merethatar inclusive: DIM A(10) -> 0..10.
  • IF: IF feltetel THEN utasitas|sorszam [ELSE ...].
  • OPEN: OPEN csatorna,eszkoz[,masodlagos][,"parancs"].
  • CMD [csatorna]: csatornara iranyitas, argumentum nelkul vissza alap kimenetre.
  • FOR: FOR V=kezdo TO veg [STEP lepes], NEXT [V].
  • GRAPHIC: mode=0..5, c=0/1, s=0..25.
  • COLOR source,color: source=0..6, color=1..16.

Tesztprogram csomag

  • Ird be: TESTPACK
  • Ez 11 mintaprogramot telepit localStorage-ba.
  • Programok: TP00-INDEX, TP01-BASIC, TP02-DATA-FN, TP03-FLOW, TP04-DRAW, TP05-BOX-PAINT, TP06-CIRCLE, TP07-SPLIT, TP08-PAINT-MODE, TP09-CHANNEL, TP10-STOP-CONT.
  • Inditas: LOAD "TP00-INDEX",8 majd RUN.
  • Direkt futtatas: LOAD "TP04-DRAW",8 majd RUN.

C128 grafikai parancsok - emulalt viselkedes

  • GRAPHIC mode[,c][,s]: mode=0..5, c=0/1 (default 1), s=0..25 split modhoz.
  • GRAPHIC CLR: visszaall text modra es torli a grafikus reteget.
  • SCNCLR [mode]: text modban text torles, bitmap modban grafikai torles.
  • COLOR source,color: source=0..6, color=1..16.
  • DRAW/BOX/CIRCLE/PAINT eseten a source tartomany 0..3.
  • CIRCLE eseten javasolt explicit source, mert a legacy CIRCLE X,Y,R[,szin] [FILL] forma is megmaradt.
  • BOX esetben ha x2,y2 hianyzik, a pixelkurzor aktualis poziciojat hasznalja.
  • PAINT mode 0: flood fill; mode 1: hatterszin-terulet kitoltese.
  • GRAPHIC 5 (80 oszlop) itt mod-jelzes, a text viewport tovabbra is C64-szeru.

C64 BASIC Emulator README

This document is the web view of README.md.

Basic usage

  • Line-numbered program line: 10 PRINT "HELLO"
  • Delete a line: 10
  • List program: LIST or LIST 100-300
  • Run: RUN or RUN 100
  • Continue after STOP/BREAK: CONT
  • New program: NEW

Programming mode

  • Line-numbered input lines are saved only; they are not executed on Enter.
  • Code execution happens only with a separate RUN command.
  • SHIFT+ENTER auto-prefills the next line number (+10).
  • Screen content and program state are auto-saved to localStorage, so opening README does not lose state.

Screen navigation

  • ArrowUp / ArrowDown: move between program lines for editing.
  • If you are on the last program line, ArrowDown moves back to the bottom command line.
  • After LIST, clicking a numbered line loads it into the editor.
  • During program-line editing, clicking outside a program line returns to command line mode.
  • Pressing Enter updates edited lines in place, without duplicate display lines.
  • Alt+ArrowUp / Alt+ArrowDown: scroll in the screen buffer.
  • PageUp / PageDown: faster scroll.
  • Mouse wheel scrolling also works on the screen.

View toggle

  • MAX VIEW: bigger container area with unchanged font size.
  • NORMAL VIEW: switches back to default size.
  • Hotkey: F9.

Public deploy (Vercel)

  • Push this project to a GitHub repository.
  • In Vercel: Add New Project, then import the repository.
  • If the project is not in repo root, set Root Directory to ChatGPT Codex/C64 Emulator.
  • Framework: Other, no build command required.
  • Root URL is automatically routed to index.html via vercel.json.

AI local start (3131)

  • Run: ./start-vercel.sh
  • It starts Vercel dev on fixed port 3131.
  • Open: http://localhost:3131

Storage (local)

  • SAVE "PROG",8: save program to localStorage.
  • LOAD "PROG",8: load program.
  • LOAD "*",8,1: load last saved program.
  • VERIFY "PROG",8: compare current program with saved one.

Supported BASIC features

  • Program control: LIST, RUN, NEW, CONT, HELP, TESTPACK
  • I/O and channels: OPEN, CLOSE, CMD, PRINT#, INPUT#, GET#
  • Flow control: IF THEN ELSE, GOTO, GOSUB/RETURN, ON ... GOTO/GOSUB, FOR/NEXT, STOP, END
  • Data handling: DATA, READ, RESTORE, DIM, DEF FN, LET, CLR
  • System: POKE, WAIT, GET, SYS
  • Graphics (C128-style too): GRAPHIC, SCNCLR, COLOR, DRAW, BOX, CIRCLE, PAINT, plus PLOT, LINE, RECT, CLSG
  • Screen clear: CLS (text), CLSG (graphics)
  • AI: footer AI button generates BASIC programs from prompts.

AI BASIC generator

  • Click the bottom AI button.
  • Enter your task in the prompt (example: make a Gomoku game in BASIC).
  • The generated program is inserted into the editor and auto-saved as AI-....
  • If no API key is configured on deploy, the button shows AI OFF and is inactive.
  • The AI backend accepts only supported commands; unknown statements are rejected.
  • The AI prompt command list is built from backend whitelist, with extra statement validation on output.
  • Vercel env: OPENAI_API_KEY (required), OPENAI_MODEL (optional).

Syntax reference

  • LIST [start[-end]], RUN [line], CONT, NEW, HELP, TESTPACK
  • SAVE "NAME"[ ,device], LOAD "NAME"[ ,device][ ,mode], VERIFY "NAME"[ ,device]
  • OPEN channel,device[,secondary][,"command"], CLOSE channel, CMD [channel]
  • PRINT ..., PRINT# channel,..., INPUT ..., INPUT# ..., GET ..., GET# ...
  • IF condition THEN ... [ELSE ...], FOR ... NEXT, ON ... GOTO/GOSUB, GOTO, GOSUB, RETURN
  • READ, DATA, RESTORE [line], DIM, DEF FN, LET/A=...
  • POKE address,value, WAIT address,mask[,value], SYS addr[,A][,X][,Y], CLS, CLSG
  • GRAPHIC mode[,c][,s] or GRAPHIC CLR, SCNCLR [mode]
  • COLOR color (legacy) or COLOR source,color (C128)
  • DRAW [source],x1,y1 [TO x2,y2] ..., BOX [source],x1,y1[,x2,y2][,angle][,paint]
  • CIRCLE [source],x,y,xr[,yr][,sa][,ea][,angle][,inc], PAINT [source],x,y[,mode]
  • PLOT X,Y[,color], LINE X1,Y1,X2,Y2[,color], RECT X,Y,W,H[,color] [FILL], CIRCLE X,Y,R[,color] [FILL] (legacy)

Command validation

  • The list was verified against emulator parser paths (executeImmediate + executeStatement).
  • Immediate commands (like LIST, RUN) are separate from program-line statements.

Parameter formats

  • Variable names: numeric (A, I1) or string (N$).
  • Arrays: DIM A(10), DIM M(5,5); indexing: A(I), M(I,J).
  • DIM bounds are inclusive: DIM A(10) -> 0..10.
  • IF: IF condition THEN statement|line [ELSE ...].
  • OPEN: OPEN channel,device[,secondary][,"command"].
  • CMD [channel]: redirect output to channel; with no argument it returns to default output.
  • FOR: FOR V=start TO end [STEP step], NEXT [V].
  • GRAPHIC: mode=0..5, c=0/1, s=0..25.
  • COLOR source,color: source=0..6, color=1..16.

  • Start: LOAD "TP00-INDEX",8 then RUN.
  • Direct run: LOAD "TP04-DRAW",8 then RUN.

C128 graphics commands - emulated behavior

  • GRAPHIC mode[,c][,s]: mode=0..5, c=0/1 (default 1), s=0..25 for split mode.
  • GRAPHIC CLR: returns to text mode and clears graphics layer.
  • SCNCLR [mode]: clears text in text mode, clears graphics in bitmap modes.
  • COLOR source,color: source=0..6, color=1..16.
  • DRAW/BOX/CIRCLE/PAINT use source range 0..3.
  • For CIRCLE, explicit source is recommended because legacy CIRCLE X,Y,R[,color] [FILL] is also supported.
  • If x2,y2 is missing in BOX, current pixel cursor position is used.
  • PAINT mode 0: flood fill; mode 1: background-area fill mode.
  • GRAPHIC 5 (80 columns) is mode signaling here; text viewport remains C64-style.
